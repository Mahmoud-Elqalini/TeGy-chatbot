AI Chatbot Backend - Final Production Structure
============================================

app/
├── main.py                     # Entry point (FastAPI initialization & lifespan)
├── ai/                         # Artificial Intelligence Core
│   ├── providers/              # LLM Model Providers (Fallback Chain)
│   │   ├── base.py             # Abstract base classes for LLMs
│   │   ├── gemini_provider.py  # Google Gemini (primary)
│   │   ├── groq_provider.py    # Groq/Llama (1st fallback)
│   │   ├── openrouter_provider.py # OpenRouter (2nd fallback)
│   │   └── fallback_provider.py   # Auto-failover orchestrator
│   ├── intent_detector.py      # Rule-based + LLM hybrid classifier
│   ├── intent_router.py        # Decision layer (Routing: fast_path/llm_path)
│   ├── memory_manager.py       # Redis-based session memory & lifecycle hooks
│   ├── prompt_builder.py       # Templates for System/History/User prompts
│   ├── response_generator.py   # Token counting & simple/streaming generation
│   ├── safety.py               # Input/Output validation & AI safety settings
│   └── tool_registry.py        # Tool definition and dynamic execution
├── api/                        # API Delivery Layer
│   └── v1/
│       ├── dependencies.py     # Shared DI (Auth, DB, Redis)
│       └── routes/             # Endpoint Definitions
│           ├── auth.py         # Authentication & User Management
│           ├── chat.py         # Main Conversational Endpoints
│           ├── chat_integration.py # External System Integration
│           ├── health.py       # System Status & Monitoring
│           └── sessions.py     # Chat Session CRUD
├── core/                       # Infrastructure & Configuration
│   ├── config.py               # Environment settings (Pydantic Settings)
│   ├── exceptions.py           # Global exception definitions
│   ├── observability.py        # Structured logging & monitoring
│   └── rate_limiter.py         # Redis-based sliding window rate limiting
├── db/                         # Database Access Layer
│   ├── postgres.py             # SQLAlchemy Async engine & session
│   └── redis.py                # Redis client & key management
├── models/                     # Data Models (SQLAlchemy)
│   └── chatbot/                # Chat-specific DB entities
│       ├── message.py          # Chat message schema
│       ├── session.py          # Session metadata schema
│       └── user.py             # Chat user profile schema
├── repositories/               # Data Access Patterns (CRUD)
│   ├── chatbot_user_repo.py
│   ├── message_repo.py
│   └── session_repo.py
├── schemas/                    # Pydantic Data Validation (DTOs)
│   ├── chat.py                 # Chat request/response schemas
│   ├── chat_integration.py     # Integration specific schemas
│   └── session.py              # Session management schemas
├── services/                   # Business Logic Layer (Orchestration)
│   ├── ai_orchestrator.py      # Coordinates LLM turns & tool execution
│   ├── auth_service.py         # Identity & Access logic
│   ├── chat_integration_service.py # Integration pipeline handler
│   ├── chat_service.py         # Core message lifecycle orchestrator
│   ├── event_service.py        # External data access logic
│   ├── idempotency_service.py  # Request deduplication logic
│   ├── message_service.py      # Message persistence logic
│   └── session_service.py      # Session CRUD & history logic
└── workers/                    # Background Processing (arq/celery)
    ├── arq_jobs.py             # Job definitions
    ├── summarization_worker.py # Context compression logic
    └── sync_worker.py          # Data synchronization logic
